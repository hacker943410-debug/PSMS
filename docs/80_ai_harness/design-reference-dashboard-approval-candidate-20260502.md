## 재분석 반영

- 상단 필터는 `TextInput`/`SelectInput`의 기본 `w-full`과 페이지 폭 클래스 충돌을 제거했다. 각 컨트롤을 고정 폭 래퍼 안에 배치하여 기준 viewport에서 기간 `326px`, 매장 `220px`, 직원 `244px`로 노출한다.
- 하단 `유지만료예정고객`, `예정된 일정`은 스크롤 컨테이너를 제거하고 원본처럼 각 6행 고정 노출 + 하단 `전체 보기` 버튼으로 변경했다.
- 중앙 4개 차트는 원본의 차트 타입에 맞춰 정적 SVG로 재작성했다.
- 직원 업무 처리 현황은 스택 막대, y축, grid, 범례를 반영했다.
- 매장별 매출 실적은 가로 막대, 금액 라벨, x축 라벨을 반영했다.
- 통신사별 판매 건수는 4색 도넛, 중앙 합계, 범례를 반영했다.
- 기간별 매출 흐름은 막대 + 라인 혼합, 좌우 축, 날짜 라벨을 반영했다.
- 차트 카드 내부는 flex 높이 계산으로 바꿔 라벨과 SVG가 카드 밖으로 밀려난 뒤 잘리는 문제를 제거했다.
- 하단 테이블 본문 글자는 기준 밀도에 맞춰 `11px` 계열로 조정하고, `1586x992`, `1440x900`, `1280x800`에서 셀 overflow `0`을 확인했다.
- `1280x800`에서는 페이지 스크롤을 만들지 않기 위해 차트 카드 높이와 테이블 행 높이만 압축하고, 기준 viewport `1586x992`의 외곽 좌표는 유지했다.
